# 01. 개요 (Overview)

## 한 줄 정의

> **사이(SAI)** 는 국립중앙박물관 큐레이터의 해설을 데이터 자산으로 삼아,
> 관람객과 작품 사이를 자연어·이미지·동선으로 잇는 풀스택 도슨트입니다.

## 왜 만들었나

처음엔 단순했어요. **"한국어로 잘 동작하는 RAG 챗봇을 만들어보자"**.
그런데 데이터 풍경을 들여다 볼수록 챗봇 한 화면짜리는 너무 아쉬웠습니다.
국립중앙박물관 큐레이터들이 작성한 추천 작품 해설은:

- 작품 1점당 평균 **3,400자**의 깊이 있는 본문
- 큐레이터 실명·소속·전공이 명시된 **공인된 1차 자료**
- 공공누리 3유형으로 재배포 가능
- 그러나 박물관 사이트 안에서는 PDF처럼 "한 번 읽고 닫는" 정적 콘텐츠

이 자료가 **검색·생성·동선 추천**으로 가공되면, 관람객이 박물관에 가기 전·중·후 모두를 도와주는
실제 컴패니언이 될 수 있다는 가설이 시작이었습니다.

## 누구를 위한 것인가

세 가지 관람객 페르소나에 동시에 답합니다:

| 페르소나 | 시나리오 | 어디서 답하는가 |
|---|---|---|
| **성인 1인 관람객** | "조선 잔치 그림이 궁금해. 큐레이터 톤으로 차분히 알려줘" | `/ask` 성인 모드 |
| **어린이와 함께** | "달항아리가 뭐예요?"  '~예요' 말투, 비유 위주 | `/ask` 어린이 모드 |
| **외국인 친구** | *"What's worth seeing in 1 hour?"* 영문 + 한자 병기 | `/plan` foreign + `/ask` foreign |

세 모드는 동일한 RAG 백엔드 위에 시스템 프롬프트만 다르게 얹은 구조입니다.
같은 자료에서 같은 사실이 나오되, 읽히는 결만 달라요.

## 무엇이 다른가 — 차별화 카드 4장

대부분의 RAG 데모는 "검색 → LLM" 한 화면에서 끝납니다. **사이**는 다음을 추가했습니다:

### ① 박물관 컴패니언 정체성 (단순 챗봇 아님)
사이트가 6개 페이지로 구성되며, 사용자 여정의 단계마다 다른 도구를 제공합니다.
- 박물관 가기 **전**: `/plan` 코스 빌더, 홈의 매일 큐레이션
- **가서**: `/exhibitions` 7관 36실 + 진행중 특별전 안내
- 작품을 **마주봐서**: `/work/:id` 큐레이터 본문 + 닮은 작품
- 자유 질문: `/ask` 3-mode RAG

### ② 멀티모달 (텍스트 + 이미지)
733장 작품 사진을 CLIP으로 임베딩.
- 작품 상세 하단에 "이 작품과 닮은 작품" 6점 자동 추천
- 〈기영회도〉를 보면 다른 계회도·왕실 행사 그림이 시각적 유사도로 묶임 (91% 매치)
- 텍스트만으론 못 잡는 시각적 모티프 매핑 가능

### ③ LLM-기반 관람 코스 빌더
`/plan`은 사용자의 시간·동반자·관심사를 받아:
1. 임베딩 검색으로 후보 작품 18점 추출
2. LLM에게 후보 + 위치 메타 + 동반자 톤 가이드를 전달
3. 작품 4~7점을 골라 **층 동선 + 시간 배분 + 큐레이터 한 줄**까지 마크다운으로 출력

다른 박물관 사이트에 없는 기능. SSE 스트리밍.

### ④ 매일 다른 큐레이션
30개 테마(봄, 사군자, 불교의 빛, 잊힌 이름들 …)를 day-of-year 결정적 회전.
서버 cron 없이도 같은 날엔 같은 결과, 다음 날엔 자동 갱신.
재방문 명분 + 백엔드 단순화의 좋은 균형점.

## 이름 "사이"의 의미

브랜드명 **사이**는 한국어 "사이(in-between)" 그대로:
- 작품과 당신 **사이**
- 큐레이터의 시선과 관람객 **사이**
- 어제와 오늘 **사이**
- 한국과 세계 **사이**

헤더 로고는 두 글자 사이에 한 글자 폭의 빈 공간을 두어, 이름 자체가 컨셉의 시각화가 됩니다.

```
사  이
```
