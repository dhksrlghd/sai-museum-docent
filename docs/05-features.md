# 05. 핵심 기능 (Features)

각 기능의 **무엇을 하는가 / 왜 만들었나 / 어떻게 동작하나** 를 한 묶음으로 정리.

---

## 1) 3-Mode RAG 챗 (`/ask`)

### 무엇을
하나의 자연어 질문에 대해 **성인 / 어린이 / 외국인** 톤을 토글로 바꿔 같은 사실을 다른 결로 답합니다.

### 왜
한국어로 잘 동작하는 RAG 챗봇은 흔하지만,
**대상 독자에 따라 톤을 바꾸는 건 시연 임팩트가 큰 차별화**.
또 어린이/외국인 페르소나는 박물관의 실제 관람객 그룹이라 실용적.

### 어떻게
- 시스템 프롬프트 3종 (`src/rag.py SYSTEM_PROMPTS`)
- 모두 "자료에 없는 건 절대 만들어내지 마라" 가드레일 명시
- 출처 형식도 모드별로 다름 ("— 참고: ...", "— 알려준 사람: ...", "— Sources: ...")
- 외국인 모드는 한자/한글 음차 + 영어 의미 병기 권장 ("Giyeonghoedo (耆英會圖, painting of an elders' gathering)")

### 비교 예시 — "기영회도가 뭐야?"

**adult**:
> 〈기영회도〉는 16세기 후반 조선시대의 원로 관료들이 모인 특별한 잔치 장면을 묘사한 그림입니다. ... — 참고: 〈기영회도〉 (큐레이터: 오다연)

**kid**:
> 기영회도는 조선 시대의 그림이에요. 1584년에 70살이 넘은 할아버지 관료들이 모여서 잔치를 하는 모습이에요. ... — 알려준 사람: 〈기영회도〉의 큐레이터

**foreign**:
> Giyeonghoedo (耆英會圖) is a traditional Korean painting depicting a gathering of elders ... — Sources: <Giyeonghoedo> (curator: Oh Da-yeon)

## 2) 멀티모달 (CLIP) — "이 작품과 닮은 작품"

### 무엇을
작품 상세 페이지 하단에 **시각적으로 비슷한 작품 6점** 자동 추천.

### 왜
- 텍스트 RAG만으론 "분위기 비슷한 그림" 같은 시각적 유사도를 못 잡음
- 1,423장 작품 사진이 그냥 표시만 되고 있었음 — 검색 자원으로 활용 안 됨
- 면접 어필: "텍스트+이미지 양쪽 모달리티에 임베딩 적용한 진짜 멀티모달 RAG"

### 어떻게
1. `embed_images.py`가 작품당 최대 3장씩 (총 894장 시도, 733장 성공) CLIP-ViT-B-32로 임베딩
2. 별도 Chroma 컬렉션 `kcurator_images` (cosine, 512-dim)
3. 작품 상세 진입 시 `/api/works/{id}/similar` 호출
4. 해당 작품 첫 이미지 임베딩을 query로 → 다른 작품의 임베딩과 유사도 비교
5. 작품 단위 dedupe + 자기 자신 제외

### 검증 결과
〈기영회도〉(조선 잔치 그림) 기준 top 4:
1. 〈호조낭관계회도〉 0.913 — 또다른 계회도
2. 문효세자 보양청계병 0.895 — 왕실 행사 그림
3. 〈겸가당아집도〉 0.885 — 또다른 모임 그림
4. 동래부사접왜사도 0.884 — 외교 의례 그림

→ CLIP이 "여러 인물이 그려진 의례·행사 장면"을 정확히 클러스터링.

## 3) 관람 코스 빌더 (`/plan`)

### 무엇을
사용자가 **시간 / 동반자 / 관심사** 를 입력하면 LLM이 박물관 작품 4~7점을 골라
**층 동선 + 시간 배분 + 큐레이터 한 줄 + 마무리 인사** 까지 마크다운으로 출력.

### 왜
- 다른 박물관 사이트에 없는 기능 — 진짜 차별화
- "박물관 컴패니언" 정체성을 이름값 하게 해주는 핵심 페이지
- 구조화된 LLM 출력 + 도메인 메타(위치/시대) 활용을 보여주기 좋음

### 어떻게

```
[입력]
  ⤷ duration_min: 30 | 60 | 90 | 120
  ⤷ companion:    self | kid | foreign
  ⤷ interests:    자유 텍스트
       ↓
[1] 관심사 임베딩 → top-18 후보 retrieve (작품 단위 dedupe)
       ↓
[2] LLM 입력 = system 프롬프트 + 후보 작품 카드 18개 (제목·위치·시대·요약 180자)
       ↓
[3] LLM 출력: 도입 단락 → ## H2 → ### 작품별 H3 (위치+시간+이유) → 화살표 이동 표시 → 마무리
       ↓
[4] SSE 스트리밍 (candidates → token...→ done)
       ↓
[프론트] 마크다운-라이크 텍스트 렌더링 (h2/h3/화살표 박스/볼드)
```

### 시스템 프롬프트의 핵심 규칙
- 후보 리스트 밖의 작품은 절대 만들어내지 말 것
- 동선 최적화 (1F → 2F → 3F 순으로 묶기)
- 작품당 6~10분, 층 이동 3분 가산
- 동반자가 어린이면 시각적·짧게, 외국인이면 영어 + 한자 병기

### 동반자 톤 가이드 (시스템 프롬프트 일부)
```
self:    성인 1인. 차분한 큐레이터 톤.
kid:     '~예요/~해요' 말투, 시각적 작품 위주.
foreign: 영어로 작성, 작품명은 음차 + 한자 병기.
```

## 4) 매일 다른 큐레이션 (홈)

### 무엇을
홈 메인 섹션이 매일 다른 테마로 6점 추천.
같은 날엔 같은 결과, 다음 날엔 자동 갱신.

### 왜
- 정적 추천 12점은 두 번째 방문 때 임팩트 0
- "매일 들어와도 새로움" — 재방문 명분
- cron 인프라 없이 결정적 회전으로 같은 효과

### 어떻게
```python
# src/daily_pick.py
THEMES = [...30개...]   # 봄, 산수, 왕실, 불교, 색의 미학, ...
today_index = date.toordinal() % len(THEMES)
theme = THEMES[today_index]

# 테마 시드 쿼리로 임베딩 검색 → 추천 작품 6점
picks = picks_for_theme(theme, model, coll, thumbnails, n=6)
```

서버 메모리에 당일 결과 캐시. 자정 넘어가면 새 테마 자동 적용.

### 테마 풀 (30개)

```
봄날의 정취 / 한국의 산수 / 조선 왕실의 위엄 / 불교의 빛 / 색의 미학 /
옛 사람들의 일상 / 구석기에서 통일신라까지 / 도자기 산책 / 글씨와 그림 /
무덤에서 나온 것들 / 외국과의 만남 / 어머니의 손길 / 어린 마음으로 /
호랑이 이야기 / 고요한 풍경 / 민중의 그림 / 화려한 의궤 / 보물의 기록 /
자연과 짐승 / 백자의 단아함 / 청자의 비취 / 금속에 새긴 마음 /
부처의 미소 / 신선의 세계 / 별과 우주 / 한자와 한글 / 잊힌 이름들 /
연꽃의 향기 / 매화와 대나무 / 학문의 길
```

각 테마는 ID·한국어·영어·임베딩 시드 쿼리로 구성.

## 5) 박물관 안내 (`/exhibitions`)

### 무엇을
- 진행중인 특별·테마전 카드 5건 (포스터+기간+장소+라벨)
- 1F/2F/3F 층별로 묶인 7관 카드 (펼치면 실 목록 + 작품 수)

### 왜
- 코스 빌더가 작품 시퀀스라면, 여기는 **공간 시퀀스**
- "지금 박물관 풍경" 한눈 (특별전 + 상설 트리)
- 박물관 사이트에 분산된 정보를 한 페이지로 통합

### 어떻게
- `/api/exhibitions`가 `permanent.json` + `special.json`을 합쳐 관 트리로 변환
- 각 카드는 독립적으로 렌더링되고 펼침 상태는 컴포넌트 로컬 state

## 6) 작품 상세 (`/work/:id`)

### 무엇을
- 큰 대표 이미지 + 메타 표 (시대·재질·크기·소장번호·등급·작가)
- 큐레이터 본문 (heading/paragraph/quote/caption 4종 블록 렌더링)
- 추가 작품 사진 갤러리
- **우측 sticky AI 도슨트 패널** (해당 작품에 대해 바로 질문)
- 하단 **"이 작품과 닮은 작품" 6점 그리드** (CLIP)

### 왜
- 작품 1점이 가진 정보(본문 3,400자, 이미지 5장, 큐레이터 노트)를 모두 살리는 페이지
- 우측 AI 패널 — 본문 읽다가 궁금한 거 즉시 질문
- 닮은 작품 — 책장 옆 "비슷한 책" 추천 같은 발견 경험

### UX 디테일
- 모바일에선 우측 패널이 본문 아래로 stack (`h-[70vh]`)
- 데스크톱에선 sticky top-20 + 화면 높이 고정
- AskBox는 "이 작품에 대해 더 자세히 알려주세요"로 자동 시작 쿼리

## 7) 모바일 햄버거 메뉴

### 무엇을
md 미만 화면에서 5-tab nav가 햄버거로 접힘.

### 왜
헤더에 5개 탭 + 로고 + 부제가 좁은 화면에서 가로 폭을 초과해 깨짐.

### 어떻게
- Header.jsx에 `useState(open)` + 외부 클릭 닫기 + 라우트 변경 시 자동 닫기
- 데스크톱(md+) `<nav>` 와 모바일 dropdown `<nav>` 분리, Tailwind `hidden md:flex` / `md:hidden`
