# 09. 로드맵 / 알려진 한계

## 알려진 한계 (Known Limitations)

### 데이터
- **추천 작품 ↔ 상설 작품 매칭 16%만 성공**. 84%는 매칭 실패 (제목 표기 차이 / 비전시 작품).
  → 코스 빌더가 추천 작품 위치를 모르는 경우 "큐레이터 추천 (위치 정보 없음)"으로 표시.
- **상설 작품 643점 본문 없음**. 작품 이름만 있고 큐레이터 해설은 없어 RAG 답변에 활용 제한적.
- **이미지 임베딩 18% 실패** — 박물관 hot-link 차단으로 894장 시도 중 161장 미수집.
- **상설실 1개 (사유의 방) 소개 텍스트 비어 있음** — 페이지 자체에 본문이 거의 없어서.

### 모델 / 검색
- **CLIP은 한국어 텍스트→이미지 검색 불가**. 표준 CLIP-ViT-B-32는 영어 텍스트만.
  → multilingual 텍스트 인코더를 추가로 띄우면 가능 (~600MB 모델 추가 필요).
- **Reranking 없음**. top-k retrieval 결과 그대로 사용. cross-encoder reranking으로 정확도 더 끌어올릴 여지.
- **하이브리드 검색 없음**. BM25 + 벡터 합치면 키워드 매칭 작품(소장번호 등)도 잘 찾음.

### LLM / 응답 품질
- **Foreign 모드 출처 형식 가끔 어긋남**. LLM이 `<자료 1>` 같은 라벨을 그대로 카피하는 케이스.
- **멀티턴 대화 미지원**. 매번 단일 질문 → 단일 응답.
  → "그럼 그건 누가 그렸어?" 같은 followup 안 됨.
- **할루시네이션 가드레일은 시스템 프롬프트 의존**. RAG 컨텍스트 밖 답변을 100% 차단하진 못함.
  → 후처리 검증 단계 (생성된 답이 컨텍스트에 있는지 LLM-as-judge로 검수)는 미구현.

### UX / 운영
- **즐겨찾기 / 코스 저장 없음**. 사용자가 만든 코스를 친구한테 공유하는 URL 없음.
- **이미지 업로드 검색 없음**. CLIP 자산이 있지만 서버측 업로드 엔드포인트 미구현.
- **분석 미수집**. 어떤 질문이 많이 들어오는지, 어떤 작품이 많이 클릭되는지 모름.
- **Rate limit 없음**. 트래픽 폭주 시 OpenAI 비용 보호 장치 부재.
- **다국어 UI**. 영문 페르소나 챗만 영어, UI 자체는 한국어 고정.

### 인프라
- **HF Space 1 replica**. 동시 사용자 많으면 큐잉.
- **Cold start 30초**. 14일 무사용 시 wake up 지연.

- 가치: "사이"의 핵심 약속을 더 잘 지킴

### B. 넓이 — 새로운 sexy 기능
- VR-lite 갤러리 또는 AR 작품 식별 (둘 중 택1)
- 시간: 10~15시간
- 가치: 면접에서 "와우" 한 번 더

- Rate limit + Analytics + LLM 추상화 + GitHub Actions CI
- 시간: 6~8시간
- 가치: "데모" → "운영 가능한 서비스"로 격상

지금 시점에선 **A 추천** — 핵심 약속 강화가 가장 임팩트 큼.
